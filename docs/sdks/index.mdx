mem-dog provides native SDKs in 5 languages and adapter libraries for popular AI agent frameworks.

## Language SDKs

<CardGroup cols={3}>
  <Card title="Python" icon="python" href="/sdks/python">
    `mem-dog-client` — httpx-based, async support, 70+ methods. Simple facade with 8 high-level methods.
  </Card>
  <Card title="TypeScript" icon="js">
    Native `fetch` API, zero dependencies. Full type safety with generated interfaces.
  </Card>
  <Card title="Go" icon="golang">
    Standard library only (`net/http`). Idiomatic error handling and context support.
  </Card>
  <Card title="Rust" icon="rust">
    Async `tokio` + `reqwest`. Strongly typed request/response models with serde.
  </Card>
  <Card title="Ruby" icon="gem">
    Faraday-based client. Follows Ruby conventions with method chaining.
  </Card>
</CardGroup>

## Quick Comparison

| SDK | Package | Transport | Async | Methods |
|-----|---------|-----------|-------|---------|
| Python | `mem-dog-client` | httpx | Yes | 70+ |
| TypeScript | `@mem-dog/client` | fetch | Yes | 70+ |
| Go | `mem-dog-go` | net/http | Yes (goroutines) | 70+ |
| Rust | `mem-dog-rs` | reqwest/tokio | Yes | 70+ |
| Ruby | `mem_dog` | faraday | No | 70+ |

## Installation

<CodeGroup>
```bash Python
pip install mem-dog-client
```

```bash TypeScript
npm install @mem-dog/client
```

```bash Go
go get github.com/mem-dog/mem-dog-go
```

```bash Rust
cargo add mem-dog-rs
```

```bash Ruby
gem install mem_dog
```
</CodeGroup>

<Info>
  All SDKs mirror the same REST API surface. If a method exists in one SDK, it exists in all of them with equivalent functionality.
</Info>
