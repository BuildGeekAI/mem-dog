Smart Routing assigns the right model to each task based on complexity. Instead of sending everything to a single large model, mem-dog uses a tiered system that balances cost, speed, and quality.

## 5 Model Tiers

| Tier | Default Model | Use Case |
|------|--------------|----------|
| **Small** | Gemma3:4b | Simple classification, tagging, metadata extraction |
| **Medium** | Gemma3:12b | Summarization, entity extraction, sentiment analysis |
| **Large** | Gemma3:27b | Complex reasoning, detailed analysis, long-form output |
| **Multimodal** | Qwen3-VL | Image analysis, document OCR, visual content |
| **Omni** | Qwen3.5 | Multi-modal + complex reasoning combined |

## Fallback Chains

Each tier has a configurable fallback chain. If the primary model is unavailable, mem-dog tries the next provider automatically.

```
Ollama (local) → Ollama Cloud (remote) → Gemini (API)
```

<Info>
  Fallback chains are evaluated left-to-right. The first available model handles the request. Configure chains in **AI Studio > Routing**.
</Info>

## Per-Agent-Type Assignment

Each of the 40 sub-agents can be assigned to a specific tier. For example:

- **ClassifierAgent** → Small tier (fast, simple decisions)
- **SummaryAgent** → Medium tier (balanced quality)
- **AnalysisAgent** → Large tier (detailed output)
- **ImageAgent** → Multimodal tier (vision required)

## Per-Data-Type Overrides

Override the default tier for specific data types when certain content needs more (or less) processing power.

<CodeGroup>
```json Email Override
{
  "data_type": "email",
  "agent": "SummaryAgent",
  "tier_override": "large",
  "reason": "Emails often contain nuanced context"
}
```

```json Log Override
{
  "data_type": "log_entry",
  "agent": "ClassifierAgent",
  "tier_override": "small",
  "reason": "Structured data, simple classification"
}
```
</CodeGroup>

## 6-Layer Detection

The router determines data type using a 6-layer detection cascade:

<Steps>
  <Step title="Explicit field">Check for `data_type` field in the payload</Step>
  <Step title="LLM classifier">Use the small-tier model to classify ambiguous content</Step>
  <Step title="MIME registry">Match against known MIME types</Step>
  <Step title="URL extension">Infer type from file extension in URLs</Step>
  <Step title="Content sniffing">Analyze content structure and patterns</Step>
  <Step title="Fallback">Route to BinaryBlobAgent as a catch-all</Step>
</Steps>

<Tip>
  Set `data_type` explicitly in your payloads to skip detection layers and get faster, more predictable routing.
</Tip>
