mem-dog includes a built-in MCP (Model Context Protocol) server that exposes your knowledge base as tools for AI assistants like Claude Desktop and Cursor.

## 8 MCP Tools

| Tool | Description |
|------|-------------|
| `mem_dog_add` | Store new data items with metadata and tags |
| `mem_dog_search` | Semantic search across all stored data |
| `mem_dog_get` | Retrieve a specific item by ID |
| `mem_dog_list` | List items with filters (type, date, tags) |
| `mem_dog_delete` | Remove an item by ID |
| `mem_dog_entities` | List extracted entities (people, orgs, etc.) |
| `mem_dog_memories` | Query memories by type and scope |
| `mem_dog_chat` | RAG chat with citations |

## Claude Desktop Setup

Add to your Claude Desktop config (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "mem-dog": {
      "url": "http://localhost:8080/api/v1/mcp/sse",
      "headers": {
        "Authorization": "Bearer md_your_api_key"
      }
    }
  }
}
```

## Cursor Setup

Add to your Cursor MCP settings (`.cursor/mcp.json`):

## Example Usage

Once connected, your AI assistant can use mem-dog tools naturally:

<CodeGroup>
```text Search
User: "What did we decide about the database migration?"

Assistant uses mem_dog_search:
  query: "database migration decision"
  search_mode: "hybrid"

→ Returns relevant items with context
```

```text Store
User: "Remember that the deploy deadline is April 15th"

Assistant uses mem_dog_add:
  content: "Deploy deadline is April 15th"
  memory_type: "factual"
  tags: ["deadline", "deployment"]

→ Stored as data_01J...
```
</CodeGroup>

<Info>
  The MCP server uses SSE (Server-Sent Events) for streaming. It authenticates using the same methods as the REST API — pass a `md_*` API key or JWT in the `Authorization` header.
</Info>

<Tip>
  The MCP server respects all access control rules. Tools only return data the authenticated user is authorized to see.
</Tip>
