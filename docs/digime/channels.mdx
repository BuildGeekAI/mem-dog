## Channel Configuration

Each channel is configured in the OpenClaw node's `config.yaml`:

```yaml
channels:
  whatsapp:
    enabled: true
    phone_number: "+1234567890"
    session_path: "/data/whatsapp-session"

  telegram:
    enabled: true
    bot_token: "${TELEGRAM_BOT_TOKEN}"

  slack:
    enabled: true
    app_token: "${SLACK_APP_TOKEN}"
    bot_token: "${SLACK_BOT_TOKEN}"
```

<Info>
  Channel credentials are stored as Kubernetes secrets in the `webhook-gateway` namespace and mounted into the OpenClaw node pod.
</Info>

## Message Format

All incoming messages are normalized to a common format before skill processing:

```json
{
  "channel_type": "whatsapp",
  "peer_id": "+1234567890",
  "message_id": "msg_abc123",
  "content": "What did we discuss yesterday?",
  "timestamp": "2026-03-27T10:00:00Z",
  "attachments": []
}
```

## Adding a New Channel

<Steps>
  <Step title="Enable the adapter">
    Add the channel configuration to `config.yaml` with credentials.
  </Step>
  <Step title="Map identities">
    Register channel identities for your users. See [Multi-User](/digime/multi-user) for details.
  </Step>
  <Step title="Restart the pod">
    The OpenClaw node picks up new channels on restart.
  </Step>
</Steps>

<Tip>
  Start with one channel and expand. Each channel operates independently — adding a new one does not affect existing connections.
</Tip>
