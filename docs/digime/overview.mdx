DigiMe is an AI agent built on the OpenClaw runtime that lives inside your messaging apps. It connects to memdog's RAG system, letting you store, search, and query your knowledge base through natural conversation.

## What Is DigiMe?

DigiMe bridges the gap between your messaging apps and your data. Instead of switching to a dashboard or API, you can:

- **Ask questions** and get cited answers from your knowledge base
- **Forward messages** to automatically ingest and process them
- **Search semantically** across everything you've stored
- **Ingest data** by simply sending it in a chat

## How It Works

<Steps>
  <Step title="Message arrives">
    A user sends a message in any connected channel (WhatsApp, Slack, etc.).
  </Step>
  <Step title="Identity resolution">
    DigiMe maps the channel-specific identity (phone number, Slack ID, etc.) to a memdog `user_id`.
  </Step>
  <Step title="Skill routing">
    Based on the message content or command prefix, DigiMe routes to the appropriate skill.
  </Step>
  <Step title="memdog interaction">
    The skill calls the memdog API (ingest, search, or chat) on behalf of the resolved user.
  </Step>
  <Step title="Response">
    Results are formatted for the channel and sent back to the user in the same conversation.
  </Step>
</Steps>

## Architecture

```
User (WhatsApp/Slack/etc.)
  → OpenClaw Runtime (channel adapters)
    → DigiMe Skills (bridge/ingest/query/search)
      → memdog API (ingest, search, chat)
        → Response back to user
```

<Info>
  DigiMe is separate from the webhook pipeline. It communicates directly with the memdog API using per-user credentials, not through webhook ingestion.
</Info>
